# دليل التحديث والإصلاح الشامل — DarbStu
> آخر تحديث: 2026-04-19 | الإصدار الحالي: 3.0.2

---

## أولاً: كيف يعمل نظام التحديث

```
DarbStu يبدأ
    ↓
يتصل بـ GitHub ويقرأ version.json
    ↓
يقارن الإصدار المحلي بالإصدار الجديد
    ↓
إذا وُجد إصدار أحدث → يظهر نافذة "تحديث تلقائي"
    ↓
عند الضغط → يُنزّل ZIP من GitHub
    ↓
يستخرج ملفات .py .json .bat فقط
    ↓ (لا يمس مجلد data أو بيانات الطلاب)
يُعيد تشغيل DarbStu تلقائياً
```

**الملفات المشمولة بالتحديث:** `.py` `.txt` `.json` `.bat` `.ico` `.spec`

**الملفات المحمية (لن تُمس أبداً):**
- `data/` (بيانات الطلاب والمستخدمين والإعدادات)
- `my-whatsapp-server/`
- `absences.db`

## ثانياً: مشاكل التحديث الشائعة وحلولها

### المشكلة 1: لا تظهر نافذة التحديث رغم وجود إصدار جديد

**السبب الأكثر احتمالاً:** الإصدار المحلي في `version.json` أحدث من المسجّل أو متطابق.

**الحل:**
```cmd
:: افتح ملف version.json في مجلد DarbStu
:: تأكد أن الرقم فيه أقل من الإصدار على GitHub
notepad "C:\Users\%USERNAME%\Desktop\DarbStu\version.json"
```

تأكد أن المحتوى يبدو هكذا (الرقم أقل من الإصدار الجديد):
```json
{
  "version": "3.0.1"
}
```

إذا كان الرقم مساوياً أو أكبر → **غيّره يدوياً لرقم أقل** ثم أعد تشغيل DarbStu.

---

### المشكلة 2: خطأ SSL عند التحديث

**الأعراض:**
```
SSL: CERTIFICATE_VERIFY_FAILED
certificate verify failed: unable to get local issuer certificate
```

**السبب:** EXE المجمّع لا يحمل شهادات SSL من Windows.

**الحل:** هذا محلول تلقائياً في الإصدار 3.0.2 وما بعده. إذا ظهر:
```cmd
:: تحقق من إصدارك أولاً
type "C:\Users\%USERNAME%\Desktop\DarbStu\version.json"

:: إذا كنت على إصدار قبل 3.0.2 → نزّل التحديث يدوياً (انظر قسم التحديث اليدوي)
```

---

### المشكلة 3: التحديث يبدأ ثم يتوقف أو يتجمّد

**السبب:** اتصال بطيء أو GitHub بطيء.

**الحل:**
```cmd
:: نزّل الملف يدوياً بدل الانتظار
curl -L --insecure --max-time 180 ^
  -o "%TEMP%\DarbStu_update.zip" ^
  "https://github.com/moon15mm/DarbStu/archive/refs/heads/main.zip"

:: إذا اكتمل التنزيل، استخرجه يدوياً (انظر قسم التحديث اليدوي الكامل)
```

---

### المشكلة 4: DarbStu لا يتصل بـ GitHub لفحص التحديث

**الأعراض:** لا تظهر أي رسالة تحديث ولا رسالة "أنت على أحدث إصدار".

**فحص:**
```cmd
:: هل GitHub متاح؟
curl -s -o nul -w "%{http_code}" https://raw.githubusercontent.com/moon15mm/DarbStu/main/version.json
:: النتيجة المتوقعة: 200
```

إذا لم تكن 200:
- تحقق من اتصال الإنترنت
- تحقق من إعدادات Proxy في الشركة/المدرسة
- جرب تعطيل الـ VPN مؤقتاً

---

### المشكلة 5: التحديث نجح لكن البرنامج لم يتغير

**السبب:** EXE القديم ما زال يعمل — التحديث يُحدّث ملفات `.py` لكن الـ EXE القديم محمّل في الذاكرة.

**الحل:**
```cmd
:: أغلق DarbStu تماماً
taskkill /F /IM DarbStu.exe

:: انتظر 3 ثوانٍ
timeout /t 3

:: افتح DarbStu من جديد — الآن سيقرأ الـ .py الجديدة
start "" "C:\Users\%USERNAME%\Desktop\DarbStu\DarbStu.exe"
```

> **ملاحظة تقنية:** DarbStu.exe يضيف مجلده في `sys.path` ليقرأ ملفات `.py` المحدّثة تلقائياً عند إعادة التشغيل، دون الحاجة لإعادة بناء EXE جديد.

---

### المشكلة 6: التحديث أفسد البرنامج (خطأ بعد التحديث)

```cmd
:: 1. احتفظ بمجلد data
xcopy "C:\Users\%USERNAME%\Desktop\DarbStu\data" ^
      "C:\Users\%USERNAME%\Desktop\data_backup\" /E /I /Y

:: 2. نزّل النسخة الأخيرة الكاملة يدوياً (انظر قسم التحديث اليدوي الكامل)

:: 3. بعد إعادة التثبيت، أعد مجلد data
xcopy "C:\Users\%USERNAME%\Desktop\data_backup\" ^
      "C:\Users\%USERNAME%\Desktop\DarbStu\data\" /E /I /Y
```

## ثالثاً: التحديث اليدوي الكامل

استخدم هذه الطريقة إذا فشل التحديث التلقائي تماماً.

### الخطوة 1: احتفظ بالبيانات
```cmd
:: انسخ مجلد data إلى مكان آمن
xcopy "C:\Users\%USERNAME%\Desktop\DarbStu\data" ^
      "C:\Users\%USERNAME%\Desktop\DarbStu_data_backup\" /E /I /Y
echo تم النسخ الاحتياطي
```

### الخطوة 2: نزّل آخر نسخة من GitHub
```cmd
:: نزّل ملف ZIP
curl -L --insecure --max-time 180 ^
  -o "%TEMP%\DarbStu_new.zip" ^
  "https://github.com/moon15mm/DarbStu/archive/refs/heads/main.zip"

:: تحقق من النجاح
if exist "%TEMP%\DarbStu_new.zip" (
    echo تم التنزيل بنجاح
) else (
    echo فشل التنزيل — تحقق من الإنترنت
)
```

### الخطوة 3: استخرج الملفات الجديدة
```cmd
:: استخرج ZIP إلى مجلد مؤقت
powershell -Command "Expand-Archive -Path '%TEMP%\DarbStu_new.zip' -DestinationPath '%TEMP%\DarbStu_extracted' -Force"

:: انسخ ملفات الكود فقط (لا تنسخ مجلد data)
robocopy "%TEMP%\DarbStu_extracted\DarbStu-main" ^
         "C:\Users\%USERNAME%\Desktop\DarbStu" ^
         *.py *.txt *.bat *.ico *.spec ^
         /XD data my-whatsapp-server __pycache__ .git ^
         /S /Y

:: انسخ version.json
copy /Y "%TEMP%\DarbStu_extracted\DarbStu-main\version.json" ^
        "C:\Users\%USERNAME%\Desktop\DarbStu\version.json"

echo تم النسخ
```

### الخطوة 4: أعد تشغيل DarbStu
```cmd
taskkill /F /IM DarbStu.exe 2>nul
timeout /t 3
start "" "C:\Users\%USERNAME%\Desktop\DarbStu\DarbStu.exe"
```

### الخطوة 5: استعادة البيانات (إذا لزم)
```cmd
:: فقط إذا اختفى مجلد data
xcopy "C:\Users\%USERNAME%\Desktop\DarbStu_data_backup\" ^
      "C:\Users\%USERNAME%\Desktop\DarbStu\data\" /E /I /Y
```

## سادساً: إصلاح مشكلة 502 (السيرفر لا يستجيب)

| الخطوة | الأمر |
|--------|-------|
| **1. تشخيص سريع** | `curl http://localhost:8000/` |
| **2. هل DarbStu يعمل؟** | `tasklist \| findstr DarbStu` |
| **3. هل cloudflared يعمل؟** | `tasklist \| findstr cloudflared` |
| **4. من يحجز المنفذ 8000؟** | `netstat -ano \| findstr :8000` |
| **5. أوقف العملية الحاجزة** | `taskkill /F /PID [رقم_PID]` |
| **6. أعد تشغيل DarbStu** | افتح DarbStu.exe وانتظر 30 ثانية |
| **7. اختبر nهائي** | افتح `http://localhost:8000` في المتصفح |

للإصلاح التلقائي الكامل: **شغّل `darb_fix.bat` أو `DarbFix.exe`**
