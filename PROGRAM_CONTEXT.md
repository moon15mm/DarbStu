## 1. نظرة عامة

**DarbStu (درب)** هو نظام مدرسي متكامل لإدارة الحضور والانصراف، التواصل مع أولياء الأمور، وإدارة العمليات الإدارية (تحويلات، تعاميم، إرشاد). يتميز بالقدرة على العمل في وضعين:
1. **وضع السيرفر (Local Server):** يعمل كقاعدة بيانات رئيسية وخادم ويب.
2. **وضع السحاب (Cloud Mode):** يتصل بقاعدة بيانات عن بُعد عبر API، مما يسمح بتعدد الأجهزة والمزامنة الفورية.

**المكونات الرئيسية:**
- **Tkinter GUI** — واجهة الإدارة على سطح المكتب (تضم أكثر من 28 تبويباً).
- **FastAPI Server** — خادم ويب لاستقبال طلبات المعلمين والأولياء (المنفذ 8000).
- **Single Instance Lock** — نظام حماية يمنع تشغيل أكثر من نسخة للبرنامج مع دعم إعادة المحاولة التلقائية أثناء التحديثات.
- **CloudDBClient** — محرك مزامنة سحابي ذكي مدمج في `database.py`.
- **Node.js WhatsApp Server** — خادم محلي لإدارة إرسال الرسائل عبر مكتبة `whatsapp-web.js`.

## 3. نظام الصلاحيات المطور (RBAC)

| الدور | المفتاح | التبويبات المتاحة |
|-------|---------|-------------------|
| **مدير (admin)** | `admin` | كل شيء (None = بلا قيود) |
| **وكيل (deputy)** | `deputy` | المراقبة، السجلات، إدارة الغياب، التأخر، الأعذار، الرسائل، الواتساب، التعاميم |
| **إداري (staff)** | `staff` | المراقبة، السجلات، إدارة الغياب، الأعذار، التقارير، إدارة الطلاب، التعاميم |
| **معلم (teacher)** | `teacher` | المراقبة، تحليل النتائج، تحويل طالب، نماذج المعلم، خطابات الاستفسار، التعاميم |
| **محضر (lab)** | `lab` | المراقبة، نماذج المعلم (تحضير)، التعاميم والنشرات |
| **حارس (guard)** | `guard` | لوحة القيادة، التأخر، المراقبة الحية |

## 5. نمط Cloud Client (مرجع مهم)

في وضع العميل السحابي، جميع دوال البيانات يجب أن تتحقق من `get_cloud_client()` أولاً:

```python
def _worker():
    from database import get_cloud_client
    client = get_cloud_client()
    if client and client.is_active():
        resp = client.get("/web/api/endpoint")
        data = resp.get("data") if resp.get("ok") else local_fallback()
    else:
        data = local_fallback()
    self.root.after(0, lambda d=data: self._update_ui(d))
threading.Thread(target=_worker, daemon=True).start()
```

### 5.1 أمن عمليات الحذف (تحديث 22 أبريل)
تم إلغاء نظام منع الحذف من أجهزة العميل واستبداله بنظام **تأكيد كلمة المرور** الموحد.
- **القاعدة:** أي مستخدم (مدير أو وكيل) يمكنه الحذف من أي جهاز (سيرفر أو عميل) بشرط إدخال كلمة مرور حسابه الشخصي بنجاح.
- **التنفيذ:** يتم استدعاء `database.authenticate(username, password)` قبل تنفيذ أي عملية حذف في الواجهة.

## 13. إصلاحات وتطويرات 22 أبريل 2026 (المسائية)

### أ. تنبيهات التعاميم الآلية (Automated Circular Alerts)
- **الإضافة:** تفعيل نظام تنبيه فوري في واجهة الويب يظهر بمجرد تسجيل الدخول في حال وجود تعاميم غير مقروءة.
- **التنفيذ:** استخدام دالة `checkUnreadCirculars` في JavaScript لاستعلام API `/web/api/circulars/unread-count`.
- **التصميم:** عرض نافذة منبثقة (Modal) أنيقة تخبر المستخدم بعدد التعاميم الجديدة مع زر للانتقال المباشر.

### ب. تحسين استقرار التحديثات (Update Stability)
- **الإصلاح:** تعديل نظام `Single Instance Lock` في `main.py` و `database.py` ليشمل محاولات إعادة اتصال (Retries) بفارق 0.5 ثانية.
- **السبب:** منع فشل تشغيل البرنامج عند قيام المحدث (Updater) بإعادة تشغيل النسخة الجديدة قبل تحرير المنفذ تماماً من النسخة القديمة.

### ج. تسريع واجهة المستخدم (Performance Tuning)
- **التحسين:** إضافة نظام `_USERS_CACHE` في `constants.py` واستخدامه في `users_tab.py`.
- **النتيجة:** تقليل عدد الاستعلامات لقاعدة البيانات عند التنقل بين تبويبات الإدارة، مما أدى لسرعة استجابة فورية في عرض قوائم المستخدمين.

### د. تعريب وتطوير الأدوار (New Roles & Localization)
- **التحديث:** إضافة أدوار "إداري" و"محضر" مع تعريب كامل لجميع المسميات في نوافذ الإضافة والفلترة.
- **التخصيص:** ضبط صلاحيات كل دور لضمان عدم وصول الفئات الإدارية للوظائف الحساسة مثل (إدارة الواتساب).
