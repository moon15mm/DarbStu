## 1. نظرة عامة

**DarbStu (درب)** هو نظام مدرسي متكامل لإدارة الحضور والانصراف، التواصل مع أولياء الأمور، وإدارة العمليات الإدارية (تحويلات، تعاميم، إرشاد). يتميز بالقدرة على العمل في وضعين:
1. **وضع السيرفر (Local Server):** يعمل كقاعدة بيانات رئيسية وخادم ويب.
2. **وضع السحاب (Cloud Mode):** يتصل بقاعدة بيانات عن بُعد عبر API، مما يسمح بتعدد الأجهزة والمزامنة الفورية.

**المكونات الرئيسية:**
- **Tkinter GUI** — واجهة الإدارة على سطح المكتب (تضم أكثر من 28 تبويباً).
- **FastAPI Server** — خادم ويب لاستقبال طلبات المعلمين والأولياء (المنفذ 8000).
- **Single Instance Lock** — نظام حماية يمنع تشغيل أكثر من نسخة للبرنامج مع دعم إعادة المحاولة التلقائية أثناء التحديثات.
- **CloudDBClient** — محرك مزامنة سحابي ذكي مدمج في `database.py`.
- **Node.js WhatsApp Server** — خادم محلي لإدارة إرسال الرسائل عبر مكتبة `whatsapp-web.js`.

## 5. نمط Cloud Client (مرجع مهم)

في وضع العميل السحابي، جميع دوال البيانات يجب أن تتحقق من `get_cloud_client()` أولاً:

```python
def _worker():
    from database import get_cloud_client
    client = get_cloud_client()
    if client and client.is_active():
        resp = client.get("/web/api/endpoint")
        data = resp.get("data") if resp.get("ok") else local_fallback()
    else:
        data = local_fallback()
    self.root.after(0, lambda d=data: self._update_ui(d))
threading.Thread(target=_worker, daemon=True).start()
```

- **الحل:** حذف التبويب القديم، تحديث `loadTeacherBalance()` لاستهداف المعرفات الصحيحة (`lb-balance-card`, `lb-remaining`)، وإضافة استدعاء مستقل في `showTab`.

- **السبب:** دالة `get_teachers_points_usage()` معرّفة مرتين في `database.py` — الأخيرة ترجع مفاتيح `consumed`/`total_limit`، لكن الـ API كانت تقرأ `used`/`limit` مما يسبب KeyError والرجوع للقيمة الافتراضية 100.
- **الحل:** استخدام `.get()` بمفاتيح احتياطية في `api_teacher_balance`، وجعل عرض الحد ديناميكياً عبر عنصر `lb-limit-val`.

### د. إصلاح عاجل — SyntaxError في f-string (v3.2.8)
- **المشكلة:** `SyntaxError: f-string expression part cannot include a backslash` يمنع تشغيل الخادم بعد التحديث.
- **السبب:** وجود `\'` داخل expressions الـ f-string في سطرين (بانر التنبيهات الذكية) — وهذا غير مدعوم في Python 3.11.
- **الحل:** تحويل السطرين المشكلتين إلى متغيرات مسبقة الحساب (`_alert_referral_html`, `_alert_circs_html`) قبل بداية الـ f-string، ثم الإشارة إليها داخله بـ `{_alert_referral_html}`.
- **قاعدة مهمة:** في Python < 3.12، لا يجوز أبداً وضع `\'` أو أي backslash داخل `{...}` في f-string. الحل دائماً هو حساب القيمة مسبقاً في متغير خارج الـ f-string.
